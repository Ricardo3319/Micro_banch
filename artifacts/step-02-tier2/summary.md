## Mechanism Analysis

### W1 vs W2 关键差异

| 维度 | W1 (Poisson) | W2 (MMPP+热节点) |
|------|-------------|------------------|
| 到达过程 | 平滑 Poisson | MMPP 二状态，burst=1.5× |
| 局部过载 | 无（均匀分布） | 16 热节点吸收 50% burst 流量 |
| M0 vs B2 P99 改善 | 最大 +1.8% (rho=0.70) | +65.9% (rho=0.50), +40.1% (rho=0.85) |
| M0 P999 优势 | rho=0.70 (+15.5%) 单点 | 多点显著改善 |

### 为什么 M0 在 W1 下优势消失

1. **无局部化热点**: Poisson 到达均匀分布到 64 hosts，Power-of-2 dispatch 已近似最优
2. **stale view 偏差小**: 无 MMPP 突发，负载变化平缓，10us SYNC 周期足以跟踪
3. **迁移成本/收益比恶化**: 迁移的 3.15us 网络惩罚在仅有温和排队时性价比低
4. **预算竞争**: M0 在 rho≥0.70 命中 4.5% 有效预算上限，但可迁移任务收益有限

### M0 在 rho=0.70 的 P999 优势 (+15.5%)

即使在 W1 下，M0 的预测扫描仍能在偶发的队列不均衡中识别极端尾部任务（bimodal 长任务 100us 恰好排在多个长任务之后），通过主动迁移缩短这些极端案例的等待时间。此效应仅在 P999（比 P99 更极端的尾部）上显现。

### B0 退化分析

B0 (Ideal-cFCFS) 在 rho≥0.87 退化因为：
- 全局队列拉取模型：每次核心完成任务后需 3.15us 网络延迟获取下一个任务
- 有效服务时间 = E/C + T_host + T_net_pull = 24 + 2.1 + 3.15 = 29.25us
- 最大吞吐 = 1024 / 29.25 ≈ 35.0 req/us，而 rho=0.90 需要 38.4 req/us → 过载
- 这验证了指导文档的预期："cFCFS 的通信开销将...导致吞吐量崩溃"

### 负例与退化场景

1. **M0 at rho=0.95**: P999=2550us，远超 B2 的 1600us (−59.4%)
   - 极端饱和下所有主机均满载，迁移找不到好目标
   - 无效迁移消耗有效预算，进一步恶化尾部延迟
2. **M0 at rho=0.75-0.90 P99**: 微弱劣于 B2（1-4%）
   - 4.5% 有效迁移预算大量用于低收益迁移，迁移开销抵消收益

## Lessons Learned

1. **M0 优势场景依赖性**: 发自 proactive migration 的改善高度依赖局部化突发（W2 热节点 + MMPP）。在平滑 Poisson 输入下，Power-of-2 dispatch 已接近最优，主动迁移的边际收益极小。
2. **B0 并非万能下界**: Ideal-cFCFS 有理论最优的队列公平性，但通信开销使其在高负载下不及分布式方案。这是 centralized vs distributed 调度的经典 tradeoff。
3. **P999 vs P99 敏感度差异**: M0 的预测扫描对极端尾部（P999）比对一般尾部（P99）更有效，因为 P999 对应的任务恰好是排队论预测模型能高置信度识别的高风险任务。
4. **W1 作为公平性基线**: W1 结果证明 baseline 实现无系统性偏差——在无突发场景下各方法性能趋近，排除了代码级不公平的可能。
