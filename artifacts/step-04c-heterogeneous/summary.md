### 机制分析

异构集群对所有方法都造成性能退化（P99 增加 78–86%），原因:
1. **慢节点热点效应**: 16 个慢节点（C=0.2）上任务执行时间 5× 放大，排队飞速积压
2. **有效算力下降**: 从 1024 降至 819.2（-20%），在相同有效 ρ 下绝对请求量较低，但慢节点成为瓶颈
3. **Power-of-k 分发局限**: B1 的入口分发仅看队列深度快照，不感知 capacity 差异，频繁将任务送往慢节点

M0 的优势来源:
- `estimate_remote_wait_stale()` 使用目标 host 的实际 capacity ($C_d$) 计算远端预期等待时间
- 三重判定中的收益约束 $W_i + E/C_{local} > \hat{W}_d + 5.25 + E/C_d + T_{margin}$ 自然偏好快节点（$C_d=1.0$）
- 在慢节点上排队的任务更容易触发违约风险约束，被更早迁出

### 异构 vs 同构饱和行为对比

在同构 W2 ρ=0.92 中（Step-04 freeze 已知负例）:
- M0 P99=2580 vs B2=902 → M0 **反劣于** B2（-186%）
- 原因: 同构下各 host 负载均衡，B2 的被动迁移够用，M0 的额外 5.25us 迁移开销在全局饱和时反噬

在异构 ρ=0.92 中:
- M0 P99=4550 vs B2=6700 → M0 仍**优于** B2（+32.1%）
- 原因: 异构环境下慢节点是确定性瓶颈，即使系统近饱和，从慢节点（C=0.2）迁移到快节点（C=1.0）仍有净收益（执行时间 5× 差异远超 5.25us 迁移开销）

**关键发现: 异构环境改变了 M0 在高 ρ 的退化模式。** 在同构中 M0 于 ρ=0.92 退化，而在异构中 M0 在全部 4 个 ρ 点均优于 B2。这是因为异构的 capacity 差异为 M0 提供了持续的迁移净收益空间。

## 10. 综合结论

1. **M0 在异构集群中全面优于 B2**: 4 个 ρ 点的 P99 改善率为 32–59%，远超 5% 门限。M0 的 capacity-aware 迁移决策在慢节点环境下优势显著。

2. **异构 vs 同构**: ρ=0.85 处 M0 的 P99 从 964（同构）退化到 1720（异构，+78%），但 M0 vs B2 的相对改善从 40.1% 略升至 42.5%。异构不削弱 M0 的相对能力。

3. **饱和行为差异**: 同构 ρ=0.92 时 M0 反劣于 B2，但异构 ρ=0.92 时 M0 仍优于 B2（+32%）。capacity 差异为 M0 持续提供迁移净收益。

4. **副作用可控**: mr ≤ 0.0456, imr ≤ 0.1675（均为中位数最大值），满足 mr≤0.05 和 imr≤0.30 的门限。

5. **回归无破坏**: 同构默认参数下 4 个关键点 P99 中位数与 Step-04 freeze 完全一致。
